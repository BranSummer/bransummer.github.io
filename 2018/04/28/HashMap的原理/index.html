HashMap的原理

Apr 28, 2018 | 基础

众所周知，HashMap是一个用于存储Key-Value键值对的集合，每一个键值对也叫做Entry(在Java8 中叫Node, 继承自Entry)。这些个键值对（Entry）分散存储在一个数组当中，这个数组就是HashMap的主干。

从源码注释上可以看出table的长度总是2的指数, 初始化的时候的值是16, 之所以选择16,是为了服务于从key映射到索引的hash算法.

从Key映射到HashMap数组的对应位置，会用到一个Hash函数：index = Hash（key）, hash函数需要保证返回的索引集尽量均匀发布, 我们自然而然地就会想到利用 key 的 hashcode 值来和 HashMap的长度来做取模运算: index = HashCode（Key） % Length. 这种方法实现简单, 但却不是最高效的方法, 在Java中的实现采用位运算的方式: index = HashCode（Key） & （Length - 1）

  1. 上面第一个hash()方法被称为 扰动函数 ,

  2. 将计算出来的hash值与(table.length - 1)进行&运算, 其效果与取模运算是一样的
